# Миграция структуры директорий

## Изменения

Структура LaTeX файлов была реорганизована для лучшей организации:

### Старая структура:
```
resume/
├── tex_en/          # Английские LaTeX файлы
└── tex_ru/          # Русские LaTeX файлы
```

### Новая структура:
```
resume/
└── tex/
    ├── en/          # Английские LaTeX файлы
    └── ru/          # Русские LaTeX файлы
```

## Что было обновлено

✅ **tasks.py** - все пути обновлены на `tex/en` и `tex/ru`
✅ **scripts/build_site.py** - использует `tex/en` для парсинга
✅ **Makefile** - обновлены пути копирования PDF
✅ **Документация** - README_TASKS.md и QUICKSTART.md

## Автоматическая миграция

Если у вас есть старые директории `tex_en` и `tex_ru`, запустите:

```bash
invoke migrate-structure
```

Эта команда:
1. Скопирует файлы из `tex_en` → `tex/en`
2. Скопирует файлы из `tex_ru` → `tex/ru`
3. Предложит удалить старые директории

## Ручная миграция

Если вы хотите мигрировать вручную:

```bash
# Создать новую структуру
mkdir -p tex/en tex/ru

# Скопировать файлы
cp -r tex_en/* tex/en/
cp -r tex_ru/* tex/ru/

# Проверить что всё работает
invoke build-site
invoke copy-pdfs

# Удалить старые директории (опционально)
rm -rf tex_en tex_ru
```

## Проверка после миграции

```bash
# Проверить структуру
tree -L 2 tex/

# Собрать сайт
invoke build-site

# Скопировать PDF
invoke copy-pdfs

# Проверить статус
invoke status
```

## Откат (если что-то пошло не так)

Если миграция прошла неудачно и вы не удалили старые директории:

```bash
# Удалить новую структуру
rm -rf tex/

# Откатить изменения в git
git checkout tasks.py scripts/build_site.py Makefile

# Использовать старые команды
make build-site
```

## Преимущества новой структуры

- 📁 Более логичная организация (язык как подпапка)
- 🌍 Легче добавить новые языки (tex/es, tex/fr, и т.д.)
- 🧹 Чище корневая директория проекта
- 📦 Лучше масштабируется для больших проектов
